# Module 02 – Setting Up My Programming Environment & My First Program

## 🎯 Objective

In this module, I'll set up my programming environment and successfully run my first program.

More importantly, I'll connect what I learned in **Module 01** (Source Code, Compiler, Interpreter, Machine Code and Operating System) with what actually happens when I run a program.

---

# 📚 Concepts to Explore

Learn about the following concepts using any learning resource you prefer (books, blogs, videos, official documentation, AI, etc.).

---

## IDE (Integrated Development Environment)

A software application that helps programmers write, organize, run and debug programs.

Examples:
- VS Code
- Visual Studio
- IntelliJ IDEA
- Eclipse
- PyCharm

> Explore the difference between an IDE and a Code Editor.

---

## Code Editor

A tool used to write source code.

Examples:
- VS Code
- Sublime Text
- Notepad++

Can a code editor run your program on its own?

---

## Compiler & Interpreter (Revisit)

In Module 01, you learned what these terms mean.

Now try to understand them in practice.

Think about questions like:

- When does the compiler/interpreter run?
- What exactly is it converting?
- Does every programming language use a compiler?
- What happens if a compiler/interpreter isn't installed?

---

## Installing a Programming Language

Install the programming language you'll be using throughout this learning journey.

Examples:
- GCC (C)
- Python
- Java
- C++

Learn:
- How to install it
- How to verify that it is installed
- How to check its version

---

## Terminal / Command Prompt

Understand:

- What is a Terminal?
- What is Command Prompt?
- Why do programmers use them?

Explore a few basic commands:

- `cd`
- `pwd`
- `ls` / `dir`
- `mkdir`
- `clear` / `cls`

You don't need to memorize them. Just understand what they do.

---

## Running a Program

Try to understand the complete journey of your program.

```
Write Code
      ↓
Save File
      ↓
Compiler / Interpreter
      ↓
Machine Code / Execution
      ↓
Operating System
      ↓
Output on Screen
```

Now relate this flow to everything you learned in Module 01.

# 💻 Practice

## Step 1 — Install Your Tools

Install:

- A programming language of your choice
- An IDE or Code Editor

Examples:

- VS Code + GCC
- VS Code + Python
- IntelliJ + Java
- Any other combination you're comfortable with

---

## Step 2 — Organize Your Repository

Inside your repository, create the following folder structure.

```
Programs/
└── 01-HelloWorld/
```

Inside the folder, you'll keep:

- Your program file
- A screenshot of the successful output

Example (C):

```
Programs/
└── 01-HelloWorld/
    ├── hello.c
    └── output.png
```

Example (Python):

```
Programs/
└── 01-HelloWorld/
    ├── hello.py
    └── output.png
```

Example (Java):

```
Programs/
└── 01-HelloWorld/
    ├── HelloWorld.java
    └── output.png
```

Use **any programming language** you prefer.

---

## Step 3 — Write Your First Program

Write a program that prints:

```
Hello, World!
```

After that, modify it to print:

- Your name
- Your favourite programming language
- One sentence explaining why you want to learn programming

Example:

```
Hello, World!

My name is RP.

My favourite programming language is Python.

I want to learn programming because I enjoy solving problems.
```

---

## Step 4 — Capture Your Output

Run your program successfully.

Take a screenshot that clearly shows:

- Your IDE or editor
- Your source code
- The output terminal/console

Save the screenshot(s) as:

```
output.png
```

inside the `Programs/01-HelloWorld/` folder.
